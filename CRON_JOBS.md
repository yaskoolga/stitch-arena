# Cron Jobs для Челленджей

## Обзор

StitchArena использует три cron job для автоматического управления системой челленджей.

## Настройка

### Переменные окружения

Добавьте в `.env` или в настройки Vercel:

```env
CRON_SECRET=your-secret-key-here
```

Этот секрет используется для верификации, что запросы приходят от Vercel Cron, а не от внешних источников.

## Cron Jobs

### 1. Generate Challenges

**Path:** `/api/cron/challenges/generate`
**Schedule:** `0 0 * * 1` (каждый понедельник в 00:00 UTC)
**Функция:** Генерирует 2-3 новых еженедельных челленджа на следующую неделю

**Логика:**
- Вычисляет дату следующего понедельника
- Проверяет, не существуют ли уже челленджи на эту неделю
- Создаёт 2 обязательных челленджа: Speed и Streak
- С вероятностью 50% добавляет Completion челлендж
- Все созданные челленджи имеют флаг `isAutoGenerated: true`

**Типы челленджей:**
- **Speed:** Вышить как можно больше крестиков за неделю
- **Streak:** Построить самую длинную серию ежедневных записей
- **Completion:** Завершить максимум проектов за неделю

**Пример ответа:**
```json
{
  "message": "Generated 3 challenges for next week",
  "startDate": "2026-02-24T00:00:00.000Z",
  "endDate": "2026-03-02T23:59:59.999Z",
  "types": ["speed", "streak", "completion"]
}
```

### 2. Close Challenges

**Path:** `/api/cron/challenges/close`
**Schedule:** `0 1 * * *` (каждый день в 01:00 UTC)
**Функция:** Закрывает завершённые челленджи и награждает победителей

**Логика:**
- Находит все активные челленджи с `endDate < now`
- Для каждого челленджа:
  - Устанавливает `isActive: false`
  - Вызывает `awardChallengeAchievements()` для награждения победителей
- Победители (топ-3) автоматически получают достижения:
  - 🏆 **CHALLENGE_WINNER** (1 место)
  - 🥉 **CHALLENGE_PODIUM** (топ-3)

**Пример ответа:**
```json
{
  "message": "Closed 2 challenges",
  "closed": 2,
  "results": [
    {
      "challengeId": "abc123",
      "title": "Weekly Speed Challenge",
      "success": true
    }
  ]
}
```

### 3. Update Leaderboards

**Path:** `/api/cron/challenges/update-leaderboards`
**Schedule:** `*/5 * * * *` (каждые 5 минут)
**Функция:** Обновляет лидерборды всех активных челленджей

**Логика:**
- Находит все активные челленджи (`startDate <= now <= endDate`)
- Для каждого челленджа:
  - Пересчитывает прогресс всех участников
  - Обновляет `currentProgress` в `ChallengeParticipant`
  - Обновляет запись в `ChallengeLeaderboard`
  - Пересчитывает ранги с учётом ties (одинаковых результатов)

**Пример ответа:**
```json
{
  "message": "Updated 2/2 challenge leaderboards",
  "challengesUpdated": 2,
  "totalParticipantsUpdated": 15,
  "results": [
    {
      "challengeId": "abc123",
      "participantsUpdated": 8,
      "success": true
    }
  ]
}
```

## Тестирование локально

### 1. Без авторизации (режим разработки)

В режиме `NODE_ENV=development` авторизация не требуется:

```bash
# Генерация челленджей
curl http://localhost:3000/api/cron/challenges/generate

# Закрытие челленджей
curl http://localhost:3000/api/cron/challenges/close

# Обновление лидербордов
curl http://localhost:3000/api/cron/challenges/update-leaderboards
```

### 2. С авторизацией (production)

В production режиме требуется `CRON_SECRET`:

```bash
curl -H "Authorization: Bearer your-secret-key-here" \
  https://your-domain.com/api/cron/challenges/generate
```

## Мониторинг

### Vercel Dashboard

После деплоя на Vercel:
1. Откройте проект в Vercel Dashboard
2. Перейдите в **Cron Jobs**
3. Просмотрите логи выполнения каждого cron job
4. Проверьте статус (success/failure)

### Логи приложения

Все cron jobs логируют свою работу в консоль:
- Успешные операции → `console.log`
- Ошибки → `console.error`

## Расписание cron

Формат: `minute hour day-of-month month day-of-week`

| Job | Schedule | Описание |
|-----|----------|----------|
| generate | `0 0 * * 1` | Каждый понедельник в 00:00 |
| close | `0 1 * * *` | Каждый день в 01:00 |
| update-leaderboards | `*/5 * * * *` | Каждые 5 минут |

## Troubleshooting

### Проблема: Cron jobs не выполняются

**Решение:**
1. Убедитесь, что `vercel.json` находится в корне проекта
2. Проверьте, что `CRON_SECRET` настроен в Vercel
3. Убедитесь, что проект задеплоен на Vercel (cron работает только в production)

### Проблема: Unauthorized (401)

**Решение:**
- В production: убедитесь, что `CRON_SECRET` совпадает в `.env` и в заголовке `Authorization`
- В development: проверьте, что `NODE_ENV=development`

### Проблема: Дублирование челленджей

**Решение:**
- Генерация проверяет наличие существующих челленджей перед созданием новых
- Если челленджи уже существуют, вернётся `{ skipped: true }`

## Безопасность

1. **CRON_SECRET** должен быть сложным и уникальным
2. В production cron endpoints доступны только с правильным токеном
3. Development режим отключает авторизацию для удобства тестирования
4. Vercel автоматически добавляет правильный токен при вызове cron jobs
