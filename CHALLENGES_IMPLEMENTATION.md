## 🗄️ Database Schema

### Challenge
```prisma
- id: String (cuid)
- type: String ("speed" | "streak" | "completion")
- title: String
- description: String?
- startDate: DateTime
- endDate: DateTime
- targetValue: Int (e.g., 10000 stitches)
- isActive: Boolean
- isAutoGenerated: Boolean
- createdAt: DateTime
```

### ChallengeParticipant
```prisma
- id: String (cuid)
- challengeId: String
- userId: String
- joinedAt: DateTime
- currentProgress: Int
- lastUpdated: DateTime
- UNIQUE(challengeId, userId)
```

### ChallengeLeaderboard
```prisma
- id: String (cuid)
- challengeId: String
- userId: String
- rank: Int
- score: Int
- updatedAt: DateTime
- UNIQUE(challengeId, userId)
```

## 🔌 API Endpoints

### `GET /api/challenges`
List all challenges with filters
- Query params: `status` (active|upcoming|past), `type` (speed|streak|completion)
- Response: `{ data: Challenge[], count: number }`

### `POST /api/challenges`
Create new challenge (admin/testing)
- Body: `{ type, title, description?, startDate, endDate, targetValue }`
- Response: `{ data: Challenge }`

### `GET /api/challenges/[id]`
Get challenge details with top 10 leaderboard
- Response: `{ data: { challenge, leaderboard, userParticipation?, userLeaderboardEntry? } }`

### `POST /api/challenges/[id]/join`
Join or leave a challenge
- Body: `{ action: "join" | "leave" }`
- Response: `{ data: ChallengeParticipant }` or `{ message: string }`

### `GET /api/challenges/[id]/leaderboard`
Full leaderboard with pagination
- Query params: `limit` (default 100, max 500), `offset` (default 0)
- Response: `{ data: LeaderboardEntry[], total: number, limit, offset }`

## 💡 Key Functions

### `calculateUserProgressForChallenge(userId, challenge)`
Calculates user's current progress based on challenge type:
- **speed**: Sum of dailyStitches in date range
- **streak**: Current consecutive days streak
- **completion**: Number of completed projects

### `updateLeaderboardEntry(challengeId, userId, score)`
Upserts leaderboard entry and updates participant progress

### `recalculateRanks(challengeId)`
Recalculates ranks for all participants (handles ties correctly)
- Example: scores [100, 100, 80] → ranks [1, 1, 3]

### `generateWeeklySpeedChallenge()`
Auto-generates weekly speed challenge (for cron)

### `updateChallengeProgressForUser(userId, log)`
**Auto-called when DailyLog is created**
Updates progress for all active challenges user participates in

## ⚠️ Important Notes

### Next.js 16 Breaking Change
Route handler params are now Promises:
```typescript
// ✅ Correct (Next.js 16)
export async function GET(
  req: Request,
  { params }: { params: Promise<{ id: string }> }
) {
  const { id } = await params;
}

// ❌ Wrong (Next.js 15)
export async function GET(
  req: Request,
  { params }: { params: { id: string } }
) {
  const id = params.id;
}
```

### Auto Progress Update
When a user creates a DailyLog, their challenge progress auto-updates:
- Hook location: `src/app/api/projects/[id]/logs/route.ts`
- Calls: `updateChallengeProgressForUser(userId, log)`
- Updates all active challenges user participates in
