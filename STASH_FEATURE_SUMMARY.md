# Функционал "Хомячья норка" (Stash)

## Обзор
Добавлен новый статус проектов "Stash" для хранения купленных, но ещё не начатых проектов вышивки.

## Изменения в БД

### Prisma Schema
1. **Поле `startedAt`** добавлено в модель `Project`
   - Тип: `DateTime?` (опциональное)
   - Хранит реальную дату начала вышивки (когда добавлены первые крестики)

2. **Статус по умолчанию** изменён на `"stash"`
   - Возможные статусы: `stash`, `in_progress`, `completed`, `paused` (ранее `abandoned`)

### Миграция
Создана миграция: `20260314170308_add_stash_status_and_started_at`
- Добавляет колонку `startedAt`
- Автоматически заполняет `startedAt` для существующих проектов на основе первого лога с `dailyStitches > 0`

## Логика работы

### Создание проекта
- По умолчанию новые проекты создаются со статусом `"stash"`
- Пользователь может выбрать любой статус в форме создания/редактирования

### Автоматический переход stash → in_progress
При добавлении первого daily log с `dailyStitches > 0`:
- Статус автоматически меняется на `"in_progress"`
- Поле `startedAt` устанавливается в дату лога
- Реализовано в: `src/app/api/projects/[id]/logs/route.ts`

### Фильтрация
Добавлен фильтр "Stash" в дашборде:
- Показывает все проекты в статусе `stash`
- Интегрирован рядом с существующими фильтрами (All, In Progress, Completed, Paused)

## Изменённые файлы

### Backend
- `prisma/schema.prisma` - схема БД
- `src/app/api/projects/[id]/logs/route.ts` - логика автоперехода статуса

### Frontend
- `src/app/dashboard/[[...userId]]/page.tsx` - добавлен фильтр
- `src/components/projects/project-form.tsx` - селектор статуса всегда видимый, дефолт "stash"

- `statusHint` - подсказка о работе статуса stash

## Использование

### Для пользователей
1. **Добавить проект в запасы:**
   - Создать проект (автоматически получит статус "Stash")
   - Или выбрать статус "Stash" вручную

2. **Начать вышивать:**
   - Просто добавить первый daily log с количеством крестиков > 0
   - Проект автоматически перейдёт в "In Progress" и установится дата старта

3. **Просмотр запасов:**
   - На дашборде нажать фильтр "Запасы"

### Для разработчиков
```bash
# Применить миграцию БД
npx prisma migrate deploy

# Или для dev окружения
npx prisma migrate dev
```

## Особенности
- Начальные крестики (initial stitches) НЕ считаются за начало проекта
- Только первый daily log с `dailyStitches > 0` переводит проект в активное состояние
- Проекты в stash можно редактировать как обычные проекты
- Фильтр stash работает для собственных и чужих дашбордов
