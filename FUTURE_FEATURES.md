# Будущие функции StitchArena

## 🔔 Социальные функции (расширение)

### Follow System (Следить за проектами)
**Приоритет: Высокий**

Возможность подписываться на проекты и получать уведомления об обновлениях.

#### Что включает:
- **База данных:**
  - Модель `ProjectFollow` (userId, projectId, createdAt)
  - Уникальный индекс на (userId, projectId)

- **API:**
  - `POST /api/projects/[id]/follow` - подписаться/отписаться
  - `GET /api/following` - список проектов, на которые подписан
  - `GET /api/feed/following` - лента обновлений от подписок

- **Функции:**
  - Кнопка "Follow" на публичных проектах (рядом с Like)
  - Уведомления при новых логах в подписанных проектах
  - Счетчик подписчиков проекта
  - Страница `/following` - проекты, за которыми следишь
  - Отдельная лента активности только от подписок

- **UI:**
  - Иконка колокольчика 🔔 для кнопки Follow
  - Badge с количеством подписчиков на странице проекта
  - Фильтр "Following" в Community Feed
  - Уведомления о новых обновлениях

#### Технические детали:
```typescript
// Модель Prisma
model ProjectFollow {
  id        String   @id @default(cuid())
  userId    String
  projectId String
  createdAt DateTime @default(now())
  user      User     @relation(fields: [userId], references: [id], onDelete: Cascade)
  project   Project  @relation(fields: [projectId], references: [id], onDelete: Cascade)

  @@unique([userId, projectId])
  @@index([projectId])
  @@index([userId])
}
```

#### Интеграция с достижениями:
- 🔔 **First Follow** - подписаться на первый проект
- 👥 **Popular** - собрать 10 подписчиков на своем проекте
- 🌟 **Influencer** - собрать 50 подписчиков

## 📱 Уведомления

### Notification System
**Приоритет: Высокий** (связано с Follow)

Система уведомлений для социальных взаимодействий.

#### Типы уведомлений:
- 💬 Новый комментарий к вашему проекту
- ❤️ Кто-то лайкнул ваш проект
- 🔔 Новый лог в подписанном проекте
- 🏆 Разблокировано новое достижение
- 🎯 Проект близок к завершению (95%+)

#### Что включает:
- **База данных:**
  - Модель `Notification` (userId, type, projectId, actorId, read, createdAt)

- **API:**
  - `GET /api/notifications` - список уведомлений
  - `POST /api/notifications/[id]/read` - отметить прочитанным
  - `POST /api/notifications/read-all` - отметить все прочитанными

- **UI:**
  - Иконка колокольчика в хедере с badge (количество непрочитанных)
  - Dropdown с последними уведомлениями
  - Страница `/notifications` с полным списком
  - Звуковые/визуальные эффекты (опционально)
